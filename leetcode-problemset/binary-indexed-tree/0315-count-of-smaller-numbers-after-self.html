# LeetCode 第 315 题：“计算右侧小于当前元素的个数”题解

题解地址：树状数组（Python 代码、Java 代码）。

说明：文本首发在力扣的题解版块，更新也会在第 1 时间在上面的网站中更新，这篇文章只是上面的文章的一个快照，您可以点击上面的链接看到其他网友对本文的评论。

传送门：计算右侧小于当前元素的个数。

给定一个整数数组 nums，按要求返回一个新数组 counts。数组 counts 有该性质： counts[i] 的值是 nums[i] 右侧小于 nums[i] 的元素的数量。

示例:

输入: [5,2,6,1] 输出: [2,1,1,0] 解释: 5 的右侧有 2 个更小的元素 (2 和 1). 2 的右侧仅有 1 个更小的元素 (1). 6 的右侧有 1 个更小的元素 (1). 1 的右侧有 0 个更小的元素.

# 树状数组（Python 代码、Java 代码）

思路分析：

“树状数组”属于高级的数据结构，现在先给出思路和参考代码。

  • 离散化：相对于数组中的值，我们其实更关心的是数组中的元素的“相对排名”，为了避免开辟多余的“树状数组”空间，我们首先对数组元素做预处理，这一步叫“离散化”；

“离散化”即得到一个排名表，对于示例输入: [5, 2, 6, 1] 得到的“排名表”为如下：

数 排名
555 333，表示 555 这个数在原始输入数组 [5, 2, 6, 1] 中的排名是 333，这个排名按照从小到大顺序。
222 222，表示 222 这个数在原始输入数组 [5, 2, 6, 1] 中的排名是 222，这个排名按照从小到大顺序。
666 444，表示 666 这个数在原始输入数组 [5, 2, 6, 1] 中的排名是 444，这个排名按照从小到大顺序。
111 111，表示 111 这个数在原始输入数组 [5, 2, 6, 1] 中的排名是 111，这个排名按照从小到大顺序。

“离散化”并不改变问题的本质，即示例输入: [5, 2, 6, 1] 如果改成它的排名表 [3, 2, 1, 4] ，得到的输出是一样的，你可以手动验证一下（因为我们只关心元素的相对大小，这一点上面也有所提及）。

我们的思路是：

从后向前填表。

因为我们关心右侧的元素，最后一个元素的右侧的元素个数为 000， 倒数第二元素的右侧的元素个数为 111，依次增大，从右边向左边看便于我们计算结果。因此我们从右边向左边看。

第 1 步：读到 111 。111 的排名是 111 ，首先先在“树状数组”的 111 的位置更新，执行的操作是 +1+1+1，很明显，在排名 111 之前肯定没有数了（查询排名在 111 之前的数有多少个），所以在结果数组的最后一个位置填 000；

第 2 步：读到 666。 666 的排名是 444，首先先在“树状数组”的 444 那个位置更新，执行的操作是 +1+1+1，接下来在“树状树组”里面执行一次查询，查询在排名 444 之前的元素个数有多少，结果是 111，所以在结果数组的倒数第 222 个位置填 000；

写到这里，你是不是已经看明白了思路。“计算右侧小于当前元素的个数”这件事情，就等价于“从右向左看”，先更新（“排名”的个数 +1+1+1，为的是后面元素的查询），然后做一次查询：“查询当前排名之前的元素的个数”，等价于“计算右侧小于当前元素的个数”。

如果对这个流程还不熟悉的朋友们，可以继续往下看，再看几个例子就不难理解了。

第 3 步：读到 222。 222 的排名是 222，首先先在“树状数组”的 222 那个位置更新，执行的操作是 +1+1+1，接下来在“树状树组”里面执行一次查询，查询在排名 222 之前的元素个数有多少，结果是 111，所以在结果数组的倒数第 333 个位置填 111；

第 4 步：读到 555。 555 的排名是 333，首先先在“树状数组”的 333 那个位置更新，执行的操作是 +1+1+1，接下来在“树状树组”里面执行一次查询，查询在排名 333 之前的元素个数有多少，结果是 222，所以在结果数组的倒数第 444 个位置填 222；

于是 [2, 1, 1, 0] 即为所求。

参考代码：

Python 代码：

from typing import List


class Solution:
    def countSmaller(self, nums: List[int]) -> List[int]:
        class FenwickTree:
            def __init__(self, n):
                self.size = n
                self.tree = [0 for _ in range(n + 1)]

            def __lowbit(self, index):
                return index & (-index)

            # 单点更新：将 index 这个位置 + 1
            def update(self, index, delta):
                # 从下到上，最多到 size，可以等于 size
                while index <= self.size:
                    self.tree[index] += delta
                    index += self.__lowbit(index)

            # 区间查询：查询小于等于 index 的元素个数
            # 查询的语义是"前缀和"
            def query(self, index):
                res = 0
                # 从上到下，最少到 1，可以等于 1
                while index > 0:
                    res += self.tree[index]
                    index -= self.__lowbit(index)
                return res

        # 特判
        size = len(nums)
        if size == 0:
            return []
        if size == 1:
            return [0]

        # 去重，方便离散化
        s = list(set(nums))

        s_len = len(s)

        # 离散化，借助堆
        import heapq
        heapq.heapify(s)

        rank_map = dict()
        rank = 1
        for _ in range(s_len):
            num = heapq.heappop(s)
            rank_map[num] = rank
            rank += 1

        fenwick_tree = FenwickTree(s_len)

        # 从后向前填表
        res = [None for _ in range(size)]
        # 从后向前填表
        for index in range(size - 1, -1, -1):
            # 1、查询排名
            rank = rank_map[nums[index]]
            # 2、在树状数组排名的那个位置 + 1
            fenwick_tree.update(rank, 1)
            # 3、查询一下小于等于“当前排名 - 1”的元素有多少
            res[index] = fenwick_tree.query(rank - 1)
        return res


if __name__ == '__main__':
    nums = [5, 2, 6, 1]
    solution = Solution()
    result = solution.countSmaller(nums)
    print(result)

Java 代码：

import java.util.ArrayList;
import java.util.Collections;
import java.util.HashMap;
import java.util.List;
import java.util.Map;
import java.util.Set;
import java.util.TreeSet;

/**
 * 使用树状数组
 */
public class Solution4 {


    public List<Integer> countSmaller(int[] nums) {
        List<Integer> res = new ArrayList<>();
        // 特判
        int len = nums.length;
        if (len == 0) {
            return res;
        }

        // 使用二分搜索树方便排序
        Set<Integer> set = new TreeSet();
        for (int i = 0; i < len; i++) {
            set.add(nums[i]);
        }

        // 排名表
        Map<Integer, Integer> map = new HashMap<>();
        int rank = 1;
        for (Integer num : set) {
            map.put(num, rank);
            rank++;
        }

        FenwickTree fenwickTree = new FenwickTree(set.size() + 1);
        // 从后向前填表
        for (int i = len - 1; i >= 0; i--) {
            // 1、查询排名
            rank = map.get(nums[i]);
            // 2、在树状数组排名的那个位置 + 1
            fenwickTree.update(rank, 1);
            // 3、查询一下小于等于“当前排名 - 1”的元素有多少
            res.add(fenwickTree.query(rank - 1));
        }
        Collections.reverse(res);
        return res;
    }


    private class FenwickTree {
        private int[] tree;
        private int len;

        public FenwickTree(int n) {
            this.len = n;
            tree = new int[n + 1];
        }

        // 单点更新：将 index 这个位置 + 1
        public void update(int i, int delta) {
            // 从下到上，最多到 size，可以等于 size
            while (i <= this.len) {
                tree[i] += delta;
                i += lowbit(i);
            }
        }


        // 区间查询：查询小于等于 index 的元素个数
        // 查询的语义是"前缀和"
        public int query(int i) {
            // 从右到左查询
            int sum = 0;
            while (i > 0) {
                sum += tree[i];
                i -= lowbit(i);
            }
            return sum;
        }

        public int lowbit(int x) {
            return x & (-x);
        }
    }


    public static void main(String[] args) {
        int[] nums = new int[]{5, 2, 6, 1};
        Solution4 solution4 = new Solution4();
        List<Integer> countSmaller = solution4.countSmaller(nums);
        System.out.println(countSmaller);
    }
}

  • 空间复杂度：O(N)O(N)O(N)。

# 树状数组“简介”

树状数组是实现了“前缀和”与“单点更新”的高效的数据结构。

下面这张图是我自己构想出来的“树状数组”的建树过程，觉得有点“前人栽树后人乘凉”的意思，大家看着乐呵乐呵。

# “树状数组”是原始数组的预处理数组

1、前缀和查询

简而言之，“前缀和”定义了一个数组从“头”开始的区间里的所有元素的和。事实上，“前缀和”知道了以后“区间和”也就知道了。

2、单点更新

“单点更新”并不关心这个数变成了什么，它关心一个数变化了多少。而“增加一个数”等价于“减去这个数的相反数”，因此，“单点更新”可以统一表达为“将某个位置的元素增加了多少”。

（上面这张图记为“图 1”）

如图 1，数组 C 是对原始数组 A 的预处理数组。在数组 A 之上构建数组 C，使之形成如上多叉树的样子，这个多叉树就是“树状数组”。

以下是两点说明：

  • “树状数组”不必建成动态的树形结构

学习过“堆”、“线段树”的朋友一定知道，使用数组就能方便地索引结点的左右孩子，这样的树形结构就不必创建成结点、指针那样的动态结构。

  • 如何执行“前缀和”查询和“单点更新”操作

如上图所示，构建好数组 C 以后，我们就可以抛弃数组 A 了。

1、前缀和查询：数组 A 有 888 个元素。如果要查询前 444 个元素的“和”（前缀和），本来应该查 A1、A2、A3、A4，然后把它们相加。有了数组 C 之后，我们只要查询 C4 即可。

2、“单点更新”操作：要更新结点 A1 的值，需要自底向上更新 C1、C2、C4、C8 的值。

以上我们对“树状数组”已经有了初步的认识。

# 理解预处理数组 C

注意：和“堆”一样，“树状数组”的 000 号索引不放置元素，从 111 号索引开始使用。从上图可以观察到，与数组 C 的某个结点有关的数组 A 的某些结点，它们的索引值之间有如下关系。

数组 C 的值由数组 A 的哪些元素而来 数组 A 的元素个数
C[1] = A[1] 111
C[2] = A[1] + A[2] 222
C[3] = A[3] 111
C[4] = A[1] + A[2] + A[3] + A[4] 444
C[5] = A[5] 111
C[6] = A[5] + A[6] 222
C[7] = A[7] 111
C[8] = A[1] + A[2] + A[3] + A[4] + A[5] + A[6] + A[7] + A[8] 888

（上面这张表记为“表 1”）

# 数组 C 的索引值与数组 A 的索引值的关系

为了避免行文啰嗦，我们将固定使用记号 iii 、jjj 、 kkk，它们的含义如下：

1、记号 iii ：预处理数组 C 的索引（十进制表示）。

2、记号 jjj ：原始数组 A​ 的索引（十进制表示）。

3、记号 kkk ：将 iii 的二进制表示从右向左数出的 000 的个数，遇到 111 则停止，记为 kkk。 注意我们只对数组 C 的索引 iii 计算。

（记号 kkk 暂时不理解没有关系，后面我们还会多次提到。）

注意到上表的最后一列“数组 A 的元素个数”，伟大的计算机科学家注意到这个数值和索引 iii 的规律如下：

将数组 C 的索引 iii 用二进制表示，从右向左数 000 的个数，遇到 111 则停止，数出的 000 的个数记为 kkk，则 2k2^k2​k​​ 是“数组 C 中的元素来自数组 A 的个数”。

下面看两个例子。

例：当 i=5i = 5i=5 时，计算 kkk。

分析：555 的二进制表示是 0000 0101（只展示低 8 位），从右向左数，第 111 个就是 111 ，因此 000 的个数是 000，此时 k=0k=0k=0，2k=12^k=12​k​​=1。

例：当 i=8i = 8i=8 时，计算 kkk。

分析：888 的二进制表示是 0000 1000（只展示低 8 位），从右向左数遇到 111 之前，遇到了 333 个 000，此时 k=3k = 3k=3 ，2k=82^k=82​k​​=8。

把数字 111 到 888 的二进制表示、kkk 、2k2^{k}2​k​​ 列成一张表如下。

索引 iii iii 的二进制表示（只展示低 8 位） kkk 2k2^{k}2​k​​
1 0000 0001 000 111
2 0000 0010 111 222
3 0000 0011 000 111
4 0000 0100 222 444
5 0000 0101 000 111
6 0000 0110 111 222
7 0000 0111 000 111
8 0000 1000 333 888

（上面这张表记为“表 2”）

我们发现，2k2^{k}2​k​​ 这一列和“表 1”的最后一列的值是相等的。2k2^k2​k​​ 是我们想要的。下面介绍一个很酷的操作，叫做 lowbit ，它可以高效地计算 2k2^k2​k​​：

{\rm lowbit}(i) = 2^k

# 使用 lowbit 计算 2k2^k2​k​​

lowbit 是这样计算 2k2^k2​k​​ 的：

lowbit(i) = i & (-i)

理解这行代码需要一些二进制和位运算的知识作为铺垫。为了便于说明，以下的二进制表示均只展示低 888 位。我们先要复习下面这个规则：

负数的二进制表示为：相应正数的二进制表示的反码 + 1。

例：将 −6-6−6 用二进制表示。

分析：666 的二进制表示为 0000 0110，先表示成反码，即 000 变 111，111 变 000，得 1111 1001，再加 111，得 1111 1010。即 −6-6−6 的二进制表示为 1111 1010。

例：计算 lowbit(6) 。

分析：由上个例题结合“与”运算的定义，把它们按照数位对齐上下写好。“与运算”的规则：同时为 111 才写 111，否则写 000。最后得到 0000 0010。

{:width="500"} {:align=center}

（上面这张图记为“图 2”）

这个二进制数表示成十进制数就是 2。建议大家多在稿纸上写几个具体的例子来计算 lowbit，进而理解为什么 {\rm lowbit}(i)=2^k。

下面给出一个直观解释：

1、如果我们直接将一个整数“按位取反”，再与原来的数做“与”运算，一定得到 000。

2、巧就巧在，负数的二进制表示上，除了要求对“按位取反”以外，还要“加” 111，在“加” 111 的过程中产生的进位数即是“将 iii 表示成二进制以后，从右向左数，遇到 111 停止时数出的 000 的个数”。

知道了 lowbit 以后，就可以很方便地执行“前缀和查询”和“单点更新”操作了。

# 使用 lowbit 实现“前缀和查询”

lowbit 可以帮助我们计算前缀和由预处理数组的哪些元素表示。

例：计算前 666 个元素的“前缀和”。

由“图 3”可以看出前 666 个元素的“前缀和” = C[6] + C[4]。

你可以验证一下：

C[6] = C[5] + A[6]、C[5] = A[5]，而 C[4] = A[1] + A[2] + A[3] + A[4]。

先看 C[6] ，lowbit(6) = 2， 6 - lowbit(6) = 4 正好是 C[6] 的上一个非叶子结点 C[4] 的索引值。

可以这样理解：

数组 C 的结点的下标表示了数组 C 的元素来自多少个数组 A 的元素，也可以理解成高度，那么上一个非叶子结点，其实就是从右边向左边画一条水平线，遇到的墙的索引值。

下面我们使用“前缀和(i)”表示前 i 个元素的“和”。

例：计算前 555 个元素的“前缀和”。

再看 C[5]，lowbit(5) = 1， 5 - lowbit(6) = 4 正好是 C[5] 的上一个非叶子结点 C[4] 的索引值，故“前缀和(5)” = C[5] + C[4]。

例：计算前 777 个元素的“前缀和”。

再看 C[7]，lowbit(7) = 1， 7 - lowbit(7) = 6 正好是 C[7] 的上一个非叶子结点 C[6] 的索引值，“前缀和(7)” = C[7] + C[6] + C[4]。

例：计算前 888 个元素的“前缀和”。

再看 C[8]，lowbit(8) = 8， 8 - lowbit(8) = 0， 0 表示没有，从“图 3”也可以看出从右边向左边画一条水平线，不会遇到的墙，故“前缀和(8)” = C[8]。

经过以上的分析，求“前缀和”的代码可以写出来了。

Java 代码：

/**
 * 查询前缀和
 *
 * @param i 前缀的最大索引，即查询区间 [0, i] 的所有元素之和
 */
public int query(int i) {
    // 从右到左查询
    int sum = 0;
    while (i > 0) {
        sum += tree[i];
        i -= lowbit(i);
    }
    return sum;
}

# “单点更新”操作

“单点更新”的方向是“从叶子结点到根结点”。

例：修改 A[3]， 分析对数组 C 产生的变化。

从“图 3”可以看出 A[3] 的父结点以及祖先结点依次是 C[3]、C[4]、C[8]，所以修改了 A[3] 以后 C[3]、C[4]、C[8] 的值也要修改。

先看 C[3] ，lowbit(3) = 1， 3 + lowbit(3) = 4 就是 C[3] 的父亲结点 C[4] 的索引值。

再看 C[4] ，lowbit(4) = 4， 4 + lowbit(4) = 8 就是 C[4] 的父亲结点 C[8] 的索引值。

也可以验证：“红色结点的索引值 + 右下角蓝色圆形结点的值 = 红色结点的双亲结点的索引值”。

下面我试图解释这个现象：333 即 001100110011，从右向左，遇到 000 放过，遇到 111 为止，给这个数位加 111，这个操作就相当于加上了一个 2k2^k2​k​​ 的二进制数，即一个 {\rm lowbit} 值，有意思的事情就发生在此时，马上就发发生了进位，得到 010001000100，即 444 的二进制表示。 接下来处理 010001000100，从右向左，从右向左，遇到 000 放过，遇到 111 为止，给这个数位加 111，同样地，这个操作就相当于加上了一个 2k2^k2​k​​ 的二进制数，即一个 {\rm lowbit} 值，可以看到，马上就发发生了进位，得到 100010001000，即 888 的二进制表示。

从上面的描述中，你可以发现，我们又在做“从右边到左边数，遇到 111 之前数出 000 的个数”这件事情了， 由此我们可以总结出规律：从已知子结点的索引 iii ，则结点 iii 的父结点的索引 {\rm parent} 的计算公式为：

{\rm parent}(i) = i + {\rm lowbit}(i)

不过我还想说明的是，这不是巧合和循环论证，这正是因为对 “从右边到左边数出 000 的个数，遇到 111 停止这件事情”的定义，使用 {\rm lowbit} 可以快速计算这件事成立，才会有的。

分析到这里“单点更新”的代码就可以马上写出来了。

Java 代码：

/**
 * 单点更新
 *
 * @param i     原始数组索引 i
 * @param delta 变化值 = 更新以后的值 - 原始值
 */
public void update(int i, int delta) {
    // 从下到上更新，注意，预处理数组，比原始数组的 len 大 1，故 预处理索引的最大值为 len
    while (i <= len) {
        tree[i] += delta;
        i += lowbit(i);
    }
}

public static int lowbit(int x) {
    return x & (-x);
}

可以看出“单点更新”和“前缀和查询操作”的代码量其实是很少的。

# 树状数组的初始化

这里要说明的是，初始化前缀和数组应该交给调用者来决定。下面是一种初始化的方式。树状数组的初始化可以通过“单点更新”来实现，因为“最最开始”的时候，数组的每个元素的值都为 000，每个都对应地加上原始数组的值，就完成了预处理数组 CCC 的创建。 这里要特别注意，update 操作的第 222 个索引值是一个变化值，而不是变化以后的值。因为我们的操作是逐层上报，汇报变更值会让我们的操作更加简单，这一点请大家反复体会。

Java 代码：

public FenwickTree(int[] nums) {
    this.len = nums.length + 1;
    tree = new int[this.len + 1];
    for (int i = 1; i <= len; i++) {
        update(i, nums[i]);
    }
}

基于以上所述，树状数组的完整代码已经可以写出来了。

Java 代码：

public class FenwickTree {

    /**
     * 预处理数组
     */
    private int[] tree;
    private int len;

    public FenwickTree(int n) {
        this.len = n;
        tree = new int[n + 1];
    }

    /**
     * 单点更新
     *
     * @param i     原始数组索引 i
     * @param delta 变化值 = 更新以后的值 - 原始值
     */
    public void update(int i, int delta) {
        // 从下到上更新，注意，预处理数组，比原始数组的 len 大 1，故 预处理索引的最大值为 len
        while (i <= len) {
            tree[i] += delta;
            i += lowbit(i);
        }
    }

    /**
     * 查询前缀和
     *
     * @param i 前缀的最大索引，即查询区间 [0, i] 的所有元素之和
     */
    public int query(int i) {
        // 从右到左查询
        int sum = 0;
        while (i > 0) {
            sum += tree[i];
            i -= lowbit(i);
        }
        return sum;
    }

    public static int lowbit(int x) {
        return x & (-x);
    }
}

Python 代码：

class FenwickTree:
    def __init__(self, n):
        self.size = n
        self.tree = [0 for _ in range(n + 1)]

    def __lowbit(self, index):
        return index & (-index)

    # 单点更新：从下到上，最多到 size，可以取等
    def update(self, index, delta):
        while index <= self.size:
            self.tree[index] += delta
            index += self.__lowbit(index)

    # 区间查询：从上到下，最少到 1，可以取等
    def query(self, index):
        res = 0
        while index > 0:
            res += self.tree[index]
            index -= self.__lowbit(index)
        return res
